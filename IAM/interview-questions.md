# AWS IAM Interview Questions

## Basic IAM

### 1. What is AWS IAM?

AWS Identity and Access Management (IAM) is a global AWS service that enables secure management of identities and permissions for accessing AWS resources. It controls who can authenticate and what actions they are authorized to perform.

---

### 2. Why is IAM considered a Global Service?

IAM resources such as users, groups, roles, and policies are managed globally within an AWS account and are not tied to a specific AWS Region.

---

### 3. What is Authentication?

Authentication is the process of verifying the identity of a user or service before granting access to AWS resources.

Examples:
- Username and Password
- Access Keys
- Multi-Factor Authentication (MFA)

---

### 4. What is Authorization?

Authorization determines what actions an authenticated identity is allowed or denied from performing on AWS resources.

Example:
A user may be authenticated successfully but still be denied permission to delete an S3 bucket.

---

### 5. What is an IAM User?

An IAM User is an individual identity within an AWS account that represents a person or application requiring long-term credentials.

Each IAM user can have:
- Password
- Access Keys
- Policies
- Group Membership

---

## IAM Groups

### 6. What is an IAM Group?

An IAM Group is a collection of IAM users that share the same permissions.

Instead of assigning permissions individually, permissions are attached to groups.

Example:

- Developers
- Admins
- Auditors
- Interns

---

### 7. Can an IAM User belong to multiple Groups?

Yes.

An IAM user can belong to multiple groups simultaneously.

AWS combines permissions from all attached groups.

---

### 8. Why should permissions be assigned to Groups instead of Users?

Assigning permissions to groups simplifies administration.

Benefits:

- Easier permission management
- Consistent access control
- Reduced configuration errors
- Better scalability

---

## IAM Policies

### 9. What is an IAM Policy?

An IAM Policy is a JSON document that defines permissions.

It specifies:

- Effect (Allow or Deny)
- Actions
- Resources
- Conditions

---

### 10. What are AWS Managed Policies?

AWS Managed Policies are predefined permission policies created and maintained by AWS.

Examples:

- AdministratorAccess
- AmazonS3FullAccess
- ReadOnlyAccess

---

### 11. What is a Customer Managed Policy?

A Customer Managed Policy is created and maintained by the AWS account owner.

Advantages:

- Fully customizable
- Version controlled
- Reusable across multiple identities

---

### 12. What is the difference between AWS Managed Policies and Customer Managed Policies?

| AWS Managed Policy | Customer Managed Policy |
|--------------------|------------------------|
| Created by AWS | Created by Customer |
| Cannot modify permissions | Fully customizable |
| Automatically updated | Managed by customer |
| Suitable for common use cases | Suitable for specific business requirements |

---

## IAM Permission Evaluation

### 13. What is the Principle of Least Privilege?

Users should receive only the minimum permissions necessary to perform their required tasks.

This reduces security risks and accidental misuse.

---

### 14. What is Explicit Deny?

An Explicit Deny always overrides any Allow.

Example:

Allow:
```
s3:*
```

Deny:
```
s3:DeleteObject
```

DeleteObject will always be denied.

---

### 15. What is Implicit Deny?

If no policy explicitly allows an action, AWS automatically denies the request.

This is called the default or implicit deny.

---

### 16. How does AWS evaluate permissions?

AWS follows this order:

1. Authentication
2. Gather applicable policies
3. Check for Explicit Deny
4. Check for Allow
5. If no Allow exists → Implicit Deny

---

## IAM Tools

### 17. What is IAM Policy Simulator?

IAM Policy Simulator is a tool that allows administrators to test IAM policies before applying them.

It helps verify:

- Allowed actions
- Denied actions
- Resource permissions

---

### 18. What is IAM Access Advisor?

Access Advisor displays which AWS services have been accessed by an IAM user, group, or role and when they were last used.

It helps identify unused permissions for security optimization.

---

## Security

### 19. Why should the Root User not be used for daily tasks?

The Root User has unrestricted permissions.

Using it for daily operations increases security risks.

Best practice:

- Enable MFA
- Use Root only for account-level tasks
- Create IAM users for regular work

---

### 20. What is Multi-Factor Authentication (MFA)?

MFA adds an additional layer of security by requiring two forms of authentication.

Typically:

- Password
- One-time verification code generated by an authenticator app or hardware device

Even if a password is compromised, MFA significantly reduces the risk of unauthorized access.
